What are SNPs?

🧬 Single Nucleotide Polymorphism
A SNP is a variation at a single position in a DNA sequence among individuals of the same species. For example, where most genomes have an A, a variant individual might carry a T. SNPs are the most common type of genetic variation.
🔬 Variant Calling
Variant calling is the process of identifying positions where the sequenced sample differs from a reference genome. BCFtools uses a Bayesian statistical model to call confident variants based on base qualities, read depth, and mapping quality.
📊 VCF Format
The Variant Call Format (VCF) is the standard output for variant data. It encodes chromosome position, reference allele, alternate allele, quality score (QUAL), and INFO/FORMAT fields per variant.
🔁 Read Alignment
BWA-MEM aligns short reads to the reference by finding maximum exact matches (MEMs), then extending them. For paired-end reads, both mates are aligned together, improving accuracy and allowing proper pair flagging.

Step-by-Step Commands

0

Download Input Data

Reference genome (SARS-CoV-2) + paired-end reads from NCBI SRA

We use the SARS-CoV-2 reference genome (GCF_009858895.2 / ASM985889v3) as our reference, and paired-end Illumina reads from accession SRR37561623 from NCBI's Sequence Read Archive.

bash — download reference genome
# Download compressed reference FASTA
wget https://ftp.ncbi.nlm.nih.gov/genomes/all/GCF/009/858/895/GCF_009858895.2_ASM985889v3/GCF_009858895.2_ASM985889v3_genomic.fna.gz

# Decompress
gunzip GCF_009858895.2_ASM985889v3/GCF_009858895.2_ASM985889v3_genomic.fna.gz

# Rename for convenience
mv GCF_009858895.2_ASM985889v3/GCF_009858895.2_ASM985889v3_genomic.fna ref.fa
bash — download paired-end reads from SRA
# Download SRA archive
prefetch SRR37561623

# Split into paired-end FASTQ files
fasterq-dump --split-files SRR37561623/SRR37561623.sra
# Output: SRR37561623_1.fastq (R1) and SRR37561623_2.fastq (R2)
ℹ️
Paired-end reads: _1.fastq contains the forward reads (R1) and _2.fastq contains the reverse reads (R2). Both files are required for proper alignment.
1

Index the Reference Genome

Build lookup structures for fast alignment and position queries

BWA indexing creates suffix array structures (BWT + SA) from the reference, enabling fast seed lookup during alignment. Samtools faidx creates a .fai index for random-access to chromosomal regions — required for mpileup.

bash — index reference
# Create BWA index (generates .amb, .ann, .bwt, .pac, .sa files)
bwa index ref.fa

# Create samtools FASTA index (generates ref.fa.fai)
samtools faidx ref.fa
⚠️
Indexing is a one-time cost per reference. For large genomes (human ~3 Gb), bwa index can take 1–2 hours. For SARS-CoV-2 (~30 kb) it completes in seconds.
2

Align Reads with BWA-MEM

Map paired-end reads to the reference genome, producing a SAM file

BWA-MEM finds maximal exact matches between reads and the reference, then chains and extends them using a Smith-Waterman alignment. The -R flag embeds a Read Group (RG) header, which is mandatory for downstream tools like GATK and recommended for bcftools.

bash — alignment
bwa mem \
  -t 8 \                        # use 8 CPU threads
  -R '@RG\tID:covid\tSM:covid\tPL:illumina' \  # read group tag
  ref.fa \                      # reference genome
  SRR37561623_1.fastq \         # forward reads (R1)
  SRR37561623_2.fastq \         # reverse reads (R2)
  > covid.sam                   # output SAM file
💡
Read Group fields: ID = unique identifier, SM = sample name (used in VCF column headers), PL = sequencing platform. These fields propagate into the final VCF for traceability.
3

Convert SAM → BAM and Sort by Query Name

Compress the alignment file and sort reads for fixmate

SAM files are plain text and large. Converting to BAM (Binary Alignment Map) reduces file size significantly. We then sort by query name (read ID) — this groups read pairs together, which is required for samtools fixmate to operate correctly.

bash — SAM to BAM + query sort
# Convert SAM to BAM (binary, compressed)
samtools view -@ 8 -bS covid.sam > covid.bam

# Sort by query/read name (-n flag)
samtools sort -@ 8 -n -o covid.querysort.bam covid.bam
⚠️
The -n (name sort) flag is essential here. Fixmate requires paired reads to be adjacent in the file. Sorting by coordinate at this stage would break fixmate.
4

Fix Mate Information

Populate mate CIGAR and insert size fields in paired-end alignments

samtools fixmate fills in the mate coordinate, mate CIGAR (MC tag), and corrects FLAG bits that describe the paired relationship. The -m flag adds the mate score tag, required by markdup.

bash — fixmate
# Fix mate pair information; -m adds mate score tag
samtools fixmate -@ 8 -m covid.querysort.bam covid.fixmate.bam
5

Re-sort by Genomic Position

Coordinate-sort the BAM for variant calling and IGV visualization

After fixmate, we re-sort the BAM by chromosomal coordinate. All downstream tools — bcftools mpileup, IGV, GATK — require coordinate-sorted input. This is the "final" alignment sort order.

bash — coordinate sort
# Sort by genomic coordinate (default sort order)
samtools sort -@ 8 -o covid.positionsort.bam covid.fixmate.bam
6

Mark PCR Duplicates

Identify and flag reads arising from PCR amplification rather than unique molecules

During library preparation, PCR amplification creates duplicate molecules from the same original DNA fragment. These duplicates cause artificially inflated read depth and can introduce false variant calls. samtools markdup flags these reads without removing them, so they can be ignored by downstream callers.

bash — mark duplicates + index
# Mark optical and PCR duplicates
samtools markdup -@ 8 covid.positionsort.bam covid.markdup.bam

# Index the final BAM (creates .bai file)
samtools index covid.markdup.bam
✅
covid.markdup.bam is your final processed alignment. You can now load it into IGV or Tablet to visually inspect read pileups over the genome.
7

Variant Calling with BCFtools

Pileup reads across the genome and call variants using a Bayesian model

bcftools mpileup summarizes base calls at each genomic position. It pipes directly into bcftools call, which applies a Bayesian genotype likelihood model to identify positions where the data supports a variant. The -mv flags select only variant sites using the multiallelic caller.

bash — pileup → variant call
bcftools mpileup \
  --threads 8 \               # parallelise
  -Ou \                       # output uncompressed BCF (pipe-friendly)
  -f ref.fa \                 # reference FASTA
  covid.markdup.bam \
| bcftools call \
  --threads 8 \
  -m \                        # multiallelic caller
  -v \                        # output variant sites only
  -Oz \                       # output compressed VCF (.vcf.gz)
  -o covid.vcf.gz

# Index the VCF
bcftools index covid.vcf.gz
💡
The -Ou flag passes uncompressed BCF between mpileup and call via a Unix pipe, avoiding disk I/O. This is faster than writing an intermediate file. -Oz then outputs bgzip-compressed VCF for the final file.
8

Extract SNPs Only

Isolate single nucleotide variants from all variant types

The raw VCF contains all variant types: SNPs, indels (insertions/deletions), MNPs, etc. We use bcftools view -v snps to retain only single nucleotide polymorphisms for focused downstream analysis.

bash — extract SNPs
# Keep only SNP-type variants
bcftools view -v snps covid.vcf.gz -Oz -o covid.snps.vcf.gz

# Index the SNP VCF
bcftools index covid.snps.vcf.gz
9

Filter High-Quality SNPs

Remove low-confidence variants using QUAL and depth thresholds

Not all called SNPs are real. Low-quality or low-depth variants are often sequencing errors or alignment artefacts. We apply two filters:

• QUAL < 30 — Phred-scaled genotype quality. QUAL=30 means 1-in-1000 probability the call is wrong.
• DP < 10 — Minimum read depth. Calls supported by fewer than 10 reads are unreliable.

bash — filter SNPs
# Exclude variants where QUAL < 30 OR read depth < 10
bcftools filter \
  -e 'QUAL<30 || DP<10' \     # expression: EXCLUDE if true
  covid.snps.vcf.gz \
  -Oz -o covid.filtered.vcf.gz

# Index the final filtered VCF
bcftools index covid.filtered.vcf.gz
✅
covid.filtered.vcf.gz is your final high-confidence SNP set. You can inspect it with bcftools stats, annotate with snpEff, or visualize variant frequencies.
💡
To quickly view your filtered SNPs: bcftools view covid.filtered.vcf.gz | grep -v "^#". Use bcftools stats covid.filtered.vcf.gz for a full summary report.

What's Next?

📊 Variant Annotation
Use snpEff or snpSift to annotate variants with gene names, functional impact (synonymous / missense / stop-gain), and population frequency from databases like ClinVar or dbSNP.
🔍 Quality Control
Run FastQC on raw reads before alignment, and MultiQC to aggregate QC across all samples. Check alignment rates with samtools flagstat covid.markdup.bam.
🧫 Multiple Samples
For multi-sample variant calling, align each sample separately, then call jointly: bcftools mpileup -f ref.fa sample1.bam sample2.bam. Joint calling improves genotype accuracy across samples.
🗺️ Phylogenetics
For SARS-CoV-2, compare your filtered SNPs against the Nextstrain / Pango lineage databases to assign a viral clade. Tools like Pangolin and Nextclade accept VCF input.
